# Pesa Barbaadi

A Flutter app to split petrol costs 50/50 with a friend. Each person logs the fuel they paid for, and the app keeps a running tally of who owes whom — synced in real time via Firestore and exportable to Excel, PDF, or CSV.

> **Pesa Barbaadi** (literally "money wasted") — because petrol money disappears faster than you can track it.

---

## Features

- **Google Sign-In** — Firebase Authentication with Google as the identity provider.
- **Shared trips** — create a trip, share the generated Trip ID with your friend; they join with the same code.
- **50/50 balance engine** — every entry is split evenly and the running balance (who owes whom) updates live.
- **Real-time sync** — both members see the same data the moment an entry is added, edited, or removed (Firestore streams).
- **Spending trends** — monthly bar chart on the History screen so you can see how spending tracks over time.
- **Multi-format export** — generate reports as Excel (`.xlsx`, grouped by month with per-sheet totals), PDF (summary + entries table), or CSV.
- **Settled detection** — balances within ₹1 of zero are treated as settled.
- **Trip persistence** — the active Trip ID is stored locally so members are dropped straight into the home screen on next launch.
- **Automated CI/CD** — every push to `main` builds a signed release APK and distributes it to Firebase App Distribution.

## Firestore data model

```
trips/{tripId}
├── members: { <uid>: <displayName>, ... }   # max 2 members
├── balance:
│   ├── owedBy:    <uid | null>
│   ├── owedTo:    <uid | null>
│   └── amount:    <double>                   # always positive, 0 = settled
│
└── entries/{entryId}
    ├── paidByUid:  <uid>
    ├── paidByName: <string>
    ├── amount:     <double>
    ├── date:       <Timestamp>
    ├── type:       "full" | "partial"
    ├── note:       <string | null>
    └── createdAt:  <Timestamp>
```

Balance is recomputed client-side by `BalanceService.compute()` from the entry list — the `balance` field on the trip doc is a cache.

## Getting started

### Prerequisites

- **Flutter** on the `stable` channel (3.44.0+ recommended)
- **Dart** 3.0+
- **Android Studio** or **Xcode** for device/emulator builds
- **Firebase CLI** (`npm i -g firebase-tools`) for the initial project setup
- **FlutterFire CLI** (`dart pub global activate flutterfire_cli`) to generate `firebase_options.dart`
- **Ruby** 3.3+ (only required for running Fastlane locally)

### Clone and install

```bash
git clone https://github.com/chiragdhunna/pesa_barbaadi.git
cd pesa_barbaadi
flutter pub get
```

### Firebase setup

1. Create a Firebase project at <https://console.firebase.google.com>.
2. Enable **Authentication → Google** as a sign-in provider.
3. Enable **Cloud Firestore** (start in production mode).
4. Deploy the security rules from `firestore.rules`:
   ```bash
   firebase deploy --only firestore:rules
   ```
5. Register your Android app in the Firebase console using the package name from `android/app/build.gradle`. Download `google-services.json` and place it at `android/app/google-services.json`.
6. Generate platform configuration:
   ```bash
   flutterfire configure
   ```
   This creates `lib/firebase_options.dart` used by `main.dart`.

### Run

```bash
flutter run
```

On first launch, sign in with Google → create a new trip or join one using a Trip ID shared by a friend.

## CI/CD & deployment

Pushes to `main` (and manual `workflow_dispatch`) trigger `.github/workflows/distribute.yml`, which:

1. Installs Flutter and runs `flutter test`.
2. Decodes secrets (`google-services.json`, signing keystore) from Base64.
3. Runs `bundle exec fastlane distribute`, which builds a release APK and uploads it to **Firebase App Distribution** under the `pesa-barbaadi-testers` group.

The Android version code is derived from `${{ github.run_number }}` so every CI build is uniquely numbered.

### Required GitHub Secrets

Add these under **Settings → Secrets and variables → Actions**:

| Secret | Description |
| :--- | :--- |
| `FIREBASE_APP_ID` | App ID from Firebase Console (Project Settings → General → Your apps). |
| `FIREBASE_TOKEN` | CI auth token from `firebase login:ci`. |
| `GOOGLE_SERVICES_JSON` | Base64-encoded contents of `android/app/google-services.json`. |
| `KEYSTORE_BASE64` | Base64-encoded contents of your release keystore (`.jks` / `.keystore`). |
| `KEY_ALIAS` | Alias of your signing key. |
| `KEY_PASSWORD` | Password for the signing key. |
| `STORE_PASSWORD` | Password for the keystore. |

### Encoding files for secrets

GitHub Secrets only accept plain text. Use Git Bash (or any POSIX shell) to produce the Base64 strings:

```bash
# google-services.json
cat android/app/google-services.json | base64 | tr -d '\n'

# release keystore
cat path/to/release.keystore | base64 | tr -d '\n'
```

On Windows PowerShell, use:

```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("android\app\google-services.json"))
```

### Local Fastlane usage

```bash
cd android
bundle install            # one-time
bundle exec fastlane test       # flutter test
bundle exec fastlane build      # flutter build apk --release
bundle exec fastlane distribute # build + upload to Firebase
```

`distribute` requires the same environment variables the workflow sets (`FIREBASE_APP_ID`, `FIREBASE_TOKEN`, `BUILD_NUMBER`, `KEY_ALIAS`, `KEY_PASSWORD`, `STORE_PASSWORD`).

### Manual workflow dispatch

From the GitHub **Actions** tab:

1. Select **Build & Distribute**.
2. Click **Run workflow** and choose a branch.
